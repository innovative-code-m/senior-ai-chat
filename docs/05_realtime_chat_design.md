# リアルタイムチャット設計

## 方針

MVPでは、SignalR を利用したリアルタイムチャットを予定します。

チャットは1つの共通チャットルームのみとし、個別DMはMVP対象外です。

## SignalR利用予定

ASP.NET Core Web API と SignalR を組み合わせ、メッセージ投稿時に接続中の参加者へ即時配信する設計を想定します。

## MVPのチャットルーム

- 共通チャットルームは1つ
- 参加者全員が同じ投稿を閲覧する
- 複数ルームや個別DMは将来拡張とする

## メッセージ投稿

1. ユーザーがカテゴリを選択する
2. テキスト本文を入力する
3. 投稿ボタンを押す
4. APIが認証状態と入力内容を検証する
5. メッセージを保存する
6. SignalRで参加者へ配信する

## メッセージ受信

- ログイン中の参加者は新規投稿をリアルタイムに受信する
- 画面再読み込み時は保存済みメッセージを取得する
- 接続が切れた場合は再接続または手動更新で復帰できるようにする

## 投稿カテゴリ

初期カテゴリ候補:

- 質問
- 使い方共有
- 体験談
- 注意喚起
- 雑談

カテゴリは、後から投稿を探しやすくし、参加者が投稿内容を理解しやすくするために使います。

## 反応ボタン

MVPでは軽量な反応ボタンを用意します。

初期候補:

- 参考になった
- 同じ疑問
- あとで試す

反応は、短い返信を書かなくても参加者の反応を表せるようにするための機能です。

## 削除済み投稿の扱い

- 管理者のみ投稿削除できる
- 物理削除ではなく削除済み状態にする方針
- 削除済み投稿は本文を非表示にする
- 画面には「この投稿は削除されました」のように表示する
- 削除日時と削除者は管理用に保存する

## 将来の複数ルーム化

将来、以下のようなルーム分割を検討できます。

- 初心者質問
- 活用例共有
- 注意喚起
- 雑談
- 管理者からのお知らせ

MVPでは運用を単純にするため、複数ルーム化は行いません。

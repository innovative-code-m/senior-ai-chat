# 画面設計

この文書は、MVPに必要な画面の初期設計メモです。

高齢者向けに、文字サイズ、操作対象の大きさ、説明の分かりやすさ、戻りやすさを優先します。

## Phase 2 実装対象

Phase 2 では、仮登録、承認待ち状態の確認、開発環境限定の管理者承認画面だけを実装対象にします。

Passkey / WebAuthn の本実装、ログイン、チャット閲覧、投稿、お知らせは Phase 2 では扱いません。

## トップ画面

| 項目 | 内容 |
| --- | --- |
| 目的 | サービスの概要を伝え、仮登録またはログインへ案内する |
| 主な表示項目 | プロジェクト概要、対象者、注意事項、仮登録ボタン、ログインボタン |
| 主な操作 | 仮登録へ進む、ログインへ進む |
| 高齢者向け配慮 | 専門用語を避け、ボタンを大きくし、次に何をするかを明確にする |

## 仮登録画面

| 項目 | 内容 |
| --- | --- |
| 目的 | 参加希望者が管理者承認のための申請を行う |
| 主な表示項目 | 氏名、メールアドレス、卒業時のクラス名、注意事項 |
| 主な操作 | 入力、申請送信、トップへ戻る |
| 高齢者向け配慮 | 入力項目を少なくし、エラー文を具体的に表示する |

### Phase 2 の入力とエラー表示

| 項目 | Phase 2 の扱い |
| --- | --- |
| 氏名 | 必須。実名登録の前提だが、リポジトリ内のサンプル値には実在人物名を使わない |
| メールアドレス | 必須。ログインID兼連絡先として扱い、大小文字差と前後空白は吸収する |
| 卒業時のクラス名 | 必須。同期生確認のために管理者だけが確認する |
| 入力エラー | 未入力、メールアドレス形式不正、文字数超過、重複登録を日本語で表示する |
| 送信成功 | `PendingApproval` として保存されたこと、管理者の確認を待つことを表示する |

## 承認待ち画面

| 項目 | 内容 |
| --- | --- |
| 目的 | 仮登録後、管理者承認を待つ状態を説明する |
| 主な表示項目 | 承認待ちメッセージ、次の流れ、状態確認結果、問い合わせ案内 |
| 主な操作 | 状態確認、トップへ戻る |
| 高齢者向け配慮 | 不安にならないよう、承認後に何が起きるかを短く説明する |

### Phase 2 の状態確認

| 状態 | 利用者向け表示 |
| --- | --- |
| `PendingApproval` | 管理者の確認待ちであることを表示する |
| `PasskeyRegistrationPending` | 承認済みで、次の Phase でパスキー登録へ進む状態であることを表示する |
| `Rejected` | 参加を許可できなかったこと、必要な場合は管理者へ個別に確認することを表示する |
| 未登録 | 入力したメールアドレスでは申請が見つからないことを表示する |

一般利用者向けの状態確認では、他者の氏名、メールアドレス、卒業時のクラス名は表示しません。

## パスキー登録画面

| 項目 | 内容 |
| --- | --- |
| 目的 | 承認済みユーザーがパスキーを登録する |
| 主な表示項目 | メールアドレス、承認状態、パスキー説明、登録ボタン、失敗時の案内 |
| 主な操作 | パスキー登録、再試行、管理者への相談案内確認 |
| 高齢者向け配慮 | パスキーを「端末の安全な本人確認」として説明し、手順を短くする |

## ログイン画面

| 項目 | 内容 |
| --- | --- |
| 目的 | 登録済みユーザーがメールアドレスとパスキーでログインする |
| 主な表示項目 | メールアドレス入力、パスキーログインボタン、再登録相談案内 |
| 主な操作 | メールアドレス入力、パスキーログイン |
| 高齢者向け配慮 | パスワード欄を表示せず、失敗時の次の行動を明確にする |

## チャット画面

| 項目 | 内容 |
| --- | --- |
| 目的 | 共通チャットルームでAIの使い方や体験を共有する |
| 主な表示項目 | メッセージ一覧、以前の投稿を読むボタン、投稿者名、投稿日時、カテゴリ、反応ボタン、投稿入力欄、お知らせリンク |
| 主な操作 | 投稿、カテゴリ選択、反応、反応取り消し、過去履歴の追加読み込み、メンバー一覧表示 |
| 高齢者向け配慮 | 入力欄と投稿ボタンを分かりやすくし、画面内の情報量を詰め込みすぎない |

## メンバー一覧画面

| 項目 | 内容 |
| --- | --- |
| 目的 | 参加者が同じ場にいるメンバーを確認する |
| 主な表示項目 | 表示名、必要最小限のプロフィール情報 |
| 主な操作 | 一覧閲覧、チャットへ戻る |
| 高齢者向け配慮 | メールアドレスなどの連絡先を一般表示せず、プライバシーを守る |

## 管理者画面

| 項目 | 内容 |
| --- | --- |
| 目的 | 管理者が参加者、投稿、お知らせ、認証復旧を管理する |
| 主な表示項目 | 承認待ちユーザー一覧、ユーザー状態、投稿管理、お知らせ管理、パスキー再登録許可 |
| 主な操作 | 承認、否認、停止、投稿削除、お知らせ投稿、再登録許可 |
| 高齢者向け配慮 | 管理操作は確認画面を挟み、誤操作を防ぐ |

### Phase 2 の管理者承認画面

Phase 2 の管理者承認画面は、Passkey / WebAuthn による本認証が未実装のため、開発環境限定のローカル検証画面として扱います。本番運用では表示・利用できる前提にしません。

| 項目 | Phase 2 の扱い |
| --- | --- |
| 表示対象 | `PendingApproval` の利用者のみ |
| 表示項目 | 氏名、メールアドレス、卒業時のクラス名、登録日時、状態 |
| 操作 | 承認、否認 |
| 承認結果 | `PasskeyRegistrationPending` に変更する |
| 否認結果 | `Rejected` に変更する |
| 誤操作対策 | 承認・否認ボタンの押下前に対象者を確認できる文脈を表示する |
| 注意表示 | 開発環境限定であり、本番運用には使わないことを管理者向けに表示する |

## お知らせ画面

| 項目 | 内容 |
| --- | --- |
| 目的 | 管理者からの案内や注意事項を表示する |
| 主な表示項目 | お知らせタイトル、本文、投稿日、投稿者 |
| 主な操作 | お知らせ閲覧、チャットへ戻る |
| 高齢者向け配慮 | 重要なお知らせを見落とさないよう、読みやすい余白と見出しにする |

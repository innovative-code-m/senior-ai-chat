# 画面設計

この文書は、MVPに必要な画面の初期設計メモです。

高齢者向けに、文字サイズ、操作対象の大きさ、説明の分かりやすさ、戻りやすさを優先します。

## トップ画面

| 項目 | 内容 |
| --- | --- |
| 目的 | サービスの概要を伝え、仮登録またはログインへ案内する |
| 主な表示項目 | プロジェクト概要、対象者、注意事項、仮登録ボタン、ログインボタン |
| 主な操作 | 仮登録へ進む、ログインへ進む |
| 高齢者向け配慮 | 専門用語を避け、ボタンを大きくし、次に何をするかを明確にする |

## 仮登録画面

| 項目 | 内容 |
| --- | --- |
| 目的 | 参加希望者が管理者承認のための申請を行う |
| 主な表示項目 | 氏名、メールアドレス、卒業時のクラス名、注意事項 |
| 主な操作 | 入力、申請送信、トップへ戻る |
| 高齢者向け配慮 | 入力項目を少なくし、エラー文を具体的に表示する |

## 承認待ち画面

| 項目 | 内容 |
| --- | --- |
| 目的 | 仮登録後、管理者承認を待つ状態を説明する |
| 主な表示項目 | 承認待ちメッセージ、次の流れ、状態確認結果、問い合わせ案内 |
| 主な操作 | 状態確認、トップへ戻る |
| 高齢者向け配慮 | 不安にならないよう、承認後に何が起きるかを短く説明する |

## パスキー登録画面

| 項目 | 内容 |
| --- | --- |
| 目的 | 承認済みユーザーがパスキーを登録する |
| 主な表示項目 | メールアドレス、承認状態、パスキー説明、登録ボタン、失敗時の案内 |
| 主な操作 | パスキー登録、再試行、管理者への相談案内確認 |
| 高齢者向け配慮 | パスキーを「端末の安全な本人確認」として説明し、手順を短くする |

## ログイン画面

| 項目 | 内容 |
| --- | --- |
| 目的 | 登録済みユーザーがメールアドレスとパスキーでログインする |
| 主な表示項目 | メールアドレス入力、パスキーログインボタン、再登録相談案内 |
| 主な操作 | メールアドレス入力、パスキーログイン |
| 高齢者向け配慮 | パスワード欄を表示せず、失敗時の次の行動を明確にする |

## チャット画面

| 項目 | 内容 |
| --- | --- |
| 目的 | 共通チャットルームでAIの使い方や体験を共有する |
| 主な表示項目 | メッセージ一覧、以前の投稿を読むボタン、投稿者名、投稿日時、カテゴリ、反応ボタン、投稿入力欄、お知らせリンク |
| 主な操作 | 投稿、カテゴリ選択、反応、反応取り消し、過去履歴の追加読み込み、メンバー一覧表示 |
| 高齢者向け配慮 | 入力欄と投稿ボタンを分かりやすくし、画面内の情報量を詰め込みすぎない |

## メンバー一覧画面

| 項目 | 内容 |
| --- | --- |
| 目的 | 参加者が同じ場にいるメンバーを確認する |
| 主な表示項目 | 表示名、必要最小限のプロフィール情報 |
| 主な操作 | 一覧閲覧、チャットへ戻る |
| 高齢者向け配慮 | メールアドレスなどの連絡先を一般表示せず、プライバシーを守る |

## 管理者画面

| 項目 | 内容 |
| --- | --- |
| 目的 | 管理者が参加者、投稿、お知らせ、認証復旧を管理する |
| 主な表示項目 | 承認待ちユーザー一覧、ユーザー状態、投稿管理、お知らせ管理、パスキー再登録許可 |
| 主な操作 | 承認、否認、停止、投稿削除、お知らせ投稿、再登録許可 |
| 高齢者向け配慮 | 管理操作は確認画面を挟み、誤操作を防ぐ |

## お知らせ画面

| 項目 | 内容 |
| --- | --- |
| 目的 | 管理者からの案内や注意事項を表示する |
| 主な表示項目 | お知らせタイトル、本文、投稿日、投稿者 |
| 主な操作 | お知らせ閲覧、チャットへ戻る |
| 高齢者向け配慮 | 重要なお知らせを見落とさないよう、読みやすい余白と見出しにする |

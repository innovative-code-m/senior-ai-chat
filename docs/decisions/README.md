# decisions

このディレクトリは、設計判断や決定事項を記録する場所です。

## 用途

- 認証方式の決定
- MVP範囲の変更
- 技術選定の理由
- 運用ルールの変更
- データ保存方針の変更

## 注意事項

- 実在の同期生情報・個人情報を含めない
- 未決事項と決定事項を明確に分ける
- 決定理由と代替案を簡潔に残す

## 決定記録

- `0001_phase1_minimum_scaffold.md`: Phase 1 の最小雛形構成
- `0002_spec_review_followup.md`: 仕様レビュー指摘への対応方針
- `0003_phase2_registration_admin_approval.md`: Phase 2 の仮登録と管理者承認の実装方針

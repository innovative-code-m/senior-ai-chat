# 要件定義

## プロジェクト概要

`senior-ai-chat` は、高齢者が信頼できる小規模コミュニティの中で、生成AIの使い方を学び合い、疑問や体験を共有するための closed グループチャット試作です。

## 背景

生成AIの利用機会は増えていますが、高齢者にとっては、使い始めるまでの心理的な抵抗、用語の分かりにくさ、ログイン操作、誤情報や詐欺への不安が障壁になりやすいです。

信頼できる同期生のような小規模な場では、初歩的な疑問を出しやすく、体験を共有しながら学ぶ流れを作れる可能性があります。

## 目的

- 高齢者が生成AIをどのように受け止めるかを観察する
- 生成AIの使い方を共有する closed なチャットを試作する
- 高齢者向けUI、認証、運用、安全性の課題を検証する
- 小規模試験運用に進むためのMVPを定義する

## コンセプト

知っている相手だけが参加する、安心してAIについて質問できる小さなチャットです。

チャットの主題は、ChatGPT などの生成AIの使い方、活用例、疑問、失敗体験、注意点の共有です。

## 想定利用者

- 高校同期生
- 今年77歳となる世代を主に想定
- AI初心者を含む
- スマートフォン利用者を主に想定
- PCブラウザ利用も許容

## 参加資格

- 高校の同期生であること
- 実名で登録すること
- メールアドレスを登録すること
- 卒業時のクラス名を登録すること
- 管理者によって承認されること

## 登録情報

| 項目 | 用途 | 注意事項 |
| --- | --- | --- |
| 氏名 | 実名登録、メンバー確認 | 個人情報。実データをリポジトリに含めない |
| メールアドレス | ログインID、連絡、承認通知 | 個人情報。サンプルでも実在アドレスを使わない |
| 卒業時のクラス名 | 同期生確認 | 個人情報に準じて扱う |
| ユーザー状態 | 承認、停止、復旧管理 | 管理者のみ変更可能 |

## 認証方式

- パスワードは使用しない
- パスキー / WebAuthn を採用する
- メールアドレスはログインIDとして使用する
- 管理者承認後にパスキーを登録する
- 端末変更時は管理者による再登録許可で復旧する
- 1ユーザーが複数パスキーを登録できる設計を想定する

## ユーザー状態

| 状態 | 概要 |
| --- | --- |
| 仮登録 | 利用者が登録申請を送信した状態 |
| 承認待ち | 管理者の確認を待っている状態 |
| パスキー未登録 | 承認済みだが認証器登録が未完了の状態 |
| 有効 | チャット利用可能な状態 |
| 停止 | 管理者により利用停止された状態 |
| 再登録許可中 | 端末変更や紛失によりパスキー再登録を許可された状態 |

## 権限

| 権限 | できること |
| --- | --- |
| 未登録利用者 | トップ画面表示、仮登録 |
| 承認待ち利用者 | 承認待ち画面表示 |
| 一般メンバー | チャット閲覧、テキスト投稿、反応、メンバー一覧閲覧 |
| 管理者 | ユーザー承認、否認、停止、投稿削除、お知らせ投稿、パスキー再登録許可 |

## チャット機能

- MVPでは1つの共通チャットルームのみ
- 投稿はテキストのみ
- 画像投稿、ファイル添付、個別DMはMVP対象外
- 投稿削除は管理者のみ実行可能
- 削除済み投稿は本文を非表示にし、削除済みであることを表示する想定

## 投稿カテゴリ

投稿時にカテゴリを選べるようにします。

初期カテゴリ候補:

- 質問
- 使い方共有
- 体験談
- 注意喚起
- 雑談

## 反応機能

MVPでは軽量な反応ボタンを用意します。

初期候補:

- 参考になった
- 同じ疑問
- あとで試す

## メンバー一覧

参加者が、同じ場に誰がいるかを確認できる画面を用意します。

表示項目は最小限にし、メールアドレスなどの連絡先は一般メンバーには表示しない方針です。

## 管理者機能

- 承認待ちユーザー一覧
- ユーザー承認
- ユーザー否認
- ユーザー停止
- 投稿削除
- お知らせ投稿
- パスキー再登録許可

## MVP範囲

MVPで実装する機能:

- 仮登録
- 管理者承認
- パスキー登録
- パスキーログイン
- チャット閲覧
- テキスト投稿
- 投稿カテゴリ
- 反応ボタン
- メンバー一覧
- 管理者による投稿削除
- 管理者からのお知らせ

MVPで実装しない機能:

- 画像投稿
- ファイル添付
- 個別DM
- 既読表示
- スタンプ
- プッシュ通知
- AI自動要約
- 外部SNS連携

## 非機能要件

- スマートフォンで読みやすく操作しやすい
- 文字サイズ、余白、ボタンサイズを高齢者向けに配慮する
- 誤操作しても戻りやすい
- パスワードを使わず、認証の説明を分かりやすくする
- 参加者以外からチャット内容が見えない
- 個人情報、投稿ログ、管理ログの扱いを明確にする
- 小規模運用で管理できる

## 運用方針

- 参加者は管理者が承認する
- 個人情報は最小限のみ扱う
- お知らせでAI利用上の注意や運用ルールを共有する
- トラブル時は管理者がユーザー停止、投稿削除、パスキー再登録許可を行う
- 小規模試験運用で利用状況と困りごとを確認する

## 決定事項

- パスワードは使用しない
- 認証は Passkey / WebAuthn を前提とする
- MVPでは1つの共通チャットルームとする
- MVPでは個別DM、画像投稿、ファイル添付を実装しない
- 実在する同期生の個人情報をリポジトリに含めない

あなたは `senior-ai-chat` リポジトリの Phase 1 を実行する開発エージェントです。

Phase 1 では、React + TypeScript のフロントエンドと ASP.NET Core Web API のバックエンドを、ローカル開発環境で起動できる最小構成として作成してください。

## 2. Phase 1 の目的

Phase 1 の目的は、アプリ本体の本格実装ではなく、今後の開発を進めるための最小雛形を作ることです。

完了時点で満たすこと:

- `src/frontend/` に React + TypeScript の最小フロントエンドがある
- `src/backend/` に ASP.NET Core Web API の最小バックエンドがある
- フロントエンドとバックエンドをローカルで起動できる
- 起動手順が README または関連ドキュメントに記載されている
- 個人情報、実在の同期生情報、秘密情報を含まない

---

## 3. 重要な制約

### 3.1 プロジェクト方針

- 高齢者が安心して使える closed グループチャットの試作であることを前提にする
- 実在する同期生の氏名、メールアドレス、クラス名、旧姓、居住地などを一切入れない
- サンプル値やテスト値が必要な場合も、実在人物に見える値は避ける
- 日本語ドキュメントを基本とする
- コードコメントを書く場合は日本語を優先する

### 3.2 認証方針

- パスワード認証を追加しない
- ログイン画面やサンプルUIにパスワード入力欄を作らない
- 認証は将来の Passkey / WebAuthn 実装を前提にする
- Phase 1 では Passkey / WebAuthn の本実装は行わない

### 3.3 MVP範囲

Phase 1 では、MVP機能の本格実装は行わないでください。

特に、以下は実装しないでください。

- 画像投稿
- ファイル添付
- 個別DM
- 既読表示
- スタンプ
- プッシュ通知
- AI自動要約
- 外部SNS連携
- データベース作成
- 本番デプロイ設定
- 実データの投入

### 3.4 Phase 1 で許可される実装範囲

Phase 1 では、次の範囲に限定してください。

- フロントエンドの起動確認用画面
- バックエンドの起動確認用 API
- 将来の SignalR 利用を見据えた最小構成またはプレースホルダー
- ローカル開発用の設定
- 起動手順、検証手順、構成説明のドキュメント更新

チャット、仮登録、管理者承認、パスキー登録などの本格的な業務機能は、後続 Phase で扱います。

## 4. 推奨する作業手順

### 4.1 現状確認

最初に以下を確認してください。

- `git status --short`
- `src/`
- `tests/`
- `docs/`
- `prompts/`

既存変更がある場合は、ユーザーの変更として扱い、勝手に戻さないでください。

### 4.2 フロントエンド雛形

`src/frontend/` に React + TypeScript の最小構成を作成してください。

推奨:

- Vite + React + TypeScript
- package name は `senior-ai-chat-frontend` など、プロジェクト意図が分かる名前
- 起動確認できる最小画面
- 高齢者向けに、文字サイズ、余白、ボタンサイズ、説明の分かりやすさを意識した初期スタイル

注意:

- Phase 1 では登録画面やチャット画面を完成させない
- パスワード欄を作らない
- 実在人物に見えるサンプルデータを入れない
- 過度なデザインやMVP外機能を追加しない

### 4.3 バックエンド雛形

`src/backend/` に ASP.NET Core Web API の最小構成を作成してください。

推奨:

- プロジェクト名は `SeniorAiChat.Api`
- 最小 API または Web API テンプレートを使用
- `/health` などの起動確認用エンドポイントを用意
- 将来の SignalR 利用を見据えた構成を検討
- DB接続、認証本実装、ユーザー管理本実装は行わない

注意:

- パスワード認証を追加しない
- データベースを作成しない
- 秘密情報を設定ファイルに入れない
- 本番用の接続文字列を追加しない

### 4.4 ローカル起動手順

以下を、`README.md`、`src/README.md`、または適切なドキュメントに記載してください。

- 必要な前提ツール
- フロントエンドのセットアップ手順
- フロントエンドの起動コマンド
- バックエンドの起動コマンド
- 起動後に確認する URL
- Phase 1 時点で未実装の範囲

手順には、実行したコマンドと実際に確認できた結果を反映してください。

### 4.5 ドキュメント更新

実装内容に合わせて、関連ドキュメントを更新してください。

最低限の候補:

- `README.md`
- `src/README.md`
- `docs/07_mvp_development_plan.md`

技術選定や構成上の判断を新たに行った場合は、必要に応じて `docs/decisions/` に決定記録を追加してください。

## 5. 検証

可能な範囲で以下を実行してください。

- フロントエンドの依存関係インストール
- フロントエンドのビルド
- フロントエンドのローカル起動確認
- バックエンドの restore/build
- バックエンドのローカル起動確認

ネットワーク制限などで依存関係取得に失敗した場合は、失敗内容を明記し、必要なら承認を求めて再実行してください。

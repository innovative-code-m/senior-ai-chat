# AGENTS.md

このリポジトリで作業するAIエージェント向けの基本方針です。

## プロジェクトの目的

`senior-ai-chat` は、高齢者が小規模で信頼できる closed グループの中で、生成AIの使い方を学び合うための試作です。

目的は単なるチャットアプリ作成ではなく、高齢者がAIをどのように受け止め、どのような場面で活用できるかを観察・検証することです。

## 現在の作業フェーズ

フェーズごとの完了、完了扱い、未着手の状態は、ルートの `ROADMAP.md` を一次記録として確認してください。

Phase 1「フロントエンドとバックエンドの最小雛形作成」は完了扱いです。

Phase 2「ユーザー仮登録と管理者承認」のローカル検証実装も完了扱いです。

Phase 3「パスキー登録とログイン」のローカル検証実装も完了扱いです。

Phase 3 では、仮登録、状態確認、開発環境限定の管理者承認、Passkey / WebAuthn 登録・ログイン、HttpOnly Cookie セッションを扱います。チャット機能、データベース接続、本番向け管理者認証は後続 Phase の対象です。

エージェントは、明示的な依頼があるまで次 Phase の本格的なアプリ実装を開始しないでください。Phase 4 以降の実装に進む場合も、先に `docs/` 配下の要件・設計文書を更新し、MVP範囲と個人情報方針に反しないことを確認します。

## 作業時の基本方針

- 高齢者向けの利用体験を前提に考える
- closed グループでの安心感と参加者の信頼を優先する
- 個人情報、健康、財産、家族関係などの繊細な話題を慎重に扱う
- 生成AIの出力を断定的な助言として扱わない
- 実装前に `docs/` 配下の要件・設計文書を更新する
- 変更時は `README.md` または関連する `docs/` を更新する
- 不明点は推測で固定せず、ドキュメント上に仮説として残す
- 日本語ドキュメントを基本とする
- コードコメントを書く場合は日本語を優先する

## 初期段階から継続する重要方針

1. プロジェクトの目的を README に明確化する
2. `AGENTS.md` にAIエージェント向け作業方針を書く
3. `docs/` 配下に要件定義・設計文書の置き場を作る
4. 実装用ディレクトリは空、または `.gitkeep` で準備する
5. GitHub公開に耐える最低限の説明を整える
6. 実在の同期生情報・個人情報は絶対に入れない

## 個人情報の扱い

このプロジェクトでは、実在する同期生の氏名、メールアドレス、クラス名、旧姓、居住地などの個人情報を、サンプルデータ・テストデータ・ドキュメントに含めてはならない。

登録情報として実名、メールアドレス、卒業時のクラス名を扱う設計であっても、リポジトリ内では実データを記載しないでください。

## MVP方針

MVP範囲内の機能は以下です。

- 仮登録
- 管理者承認
- パスキー登録
- パスキーログイン
- チャット閲覧
- テキスト投稿
- 投稿カテゴリ
- 反応ボタン
- メンバー一覧
- 管理者による投稿削除
- 管理者からのお知らせ

MVP範囲外の機能を、明示的な依頼なしに追加してはいけません。画像投稿、ファイル添付、個別DM、既読表示、スタンプ、プッシュ通知、AI自動要約、外部SNS連携はMVP対象外です。

## 認証方針

- パスワード認証を提案しない
- 認証はパスキー / WebAuthn を前提とする
- メールアドレスはログインIDとして扱う
- 管理者承認後にパスキーを登録する
- 端末変更時は管理者による再登録許可で復旧する

## 実装前に確認すること

- 参加者の範囲と招待方式
- 管理者の役割と責任
- 投稿内容の保存期間
- 個人情報を含む投稿への対応
- AI連携を行う場合の利用目的、ログ保存、外部送信範囲
- 誤情報、詐欺、健康・金融相談への注意喚起方針
- さくらインターネットへのデプロイ方式、バックアップ、HTTPS、ログ保存方針

## 推奨するドキュメント更新

設計や方針を変更する場合は、関連する `docs/` 配下の文書も更新してください。

新しい検討メモを追加する場合は、目的が分かるファイル名にしてください。

## 禁止事項

- 明示的な依頼なしに本格的なアプリ実装を開始すること
- 実在する参加者の個人情報をサンプルデータとして追加すること
- 実在の同期生の氏名、連絡先、会話内容、所属、属性情報を追加すること
- パスワード認証を前提とした設計を追加すること
- MVP範囲外の機能を勝手に実装対象へ入れること
- Reactアプリ生成、ASP.NET Coreプロジェクト生成、`npm install`、`dotnet new`、パッケージ追加、データベース作成を初期化タスクで行うこと
- 医療、法律、金融に関する断定的な助言をアプリの前提にすること
- closed グループの内容を外部公開する前提で設計すること
